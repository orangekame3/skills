# 論文ディスカッション

## 概要

既に精読済みの論文レポートに対して、対話的にディスカッションを行い、深堀りした内容をコメントとしてレポートに追記する。

**NotebookLM のように、論文の記述内容に厳密にグラウンディングされた回答を行う。**

## グラウンディングルール（最重要）

以下のルールはディスカッション中の全ての回答に適用する。

1. **論文に書かれていることだけを根拠にする** — 回答は読み込んだ論文（PDF・サマリー・レポート）の記述に基づくこと。外部知識で補完しない
2. **引用を明示する** — 回答には根拠となるセクション番号・図番号・表番号・数式番号を示す（例: 「Section 3.2 によると」「Fig. 4 に示されている通り」）
3. **記載がなければ「記載なし」と答える** — 論文に該当する記述がない場合は「この論文には記載がありません」と明確に伝える。推測で埋めない
4. **推測と事実を区別する** — 論文の記述から推論する場合は「論文の記述から推測すると」と前置きし、事実の記述と明確に区別する
5. **関連論文に言及する場合** — arq 内の他の精読済み論文を参照する場合は、どの論文のどの部分かを明示する。未精読の論文の内容には言及しない

## 手順

### 1. ノートの検索と読み込み

arq 管理下の note.md を検索する。

```bash
arq show --json $ARGUMENTS
```

JSON 出力の `has_note` が true なら `note_path` のファイルを読み込む。
false の場合は life/reads/ にフォールバック:

```
reads/*_$ARGUMENTS.md
```

どちらも見つからない場合は、先に `/arq-read $ARGUMENTS` でノートを生成するよう案内する。

### 2. 論文原文の読み込み

グラウンディングのため、レポートだけでなく**論文の原文を必ず読み込む**。

#### 2a. arq サマリーの読み込み

```bash
arq show --summary $ARGUMENTS
```

サマリーには ar5iv から抽出した構造化された要約と図への参照が含まれる。

#### 2b. PDF の読み込み

```bash
arq path $ARGUMENTS
```

返されたパスから Read ツールで PDF を読み込む。長い場合はページ範囲を指定して分割読み込み。
**ディスカッション中にユーザーが特定のセクションや図について質問した場合、該当ページを再度読み込んで正確に回答する。**

サマリーが存在しない場合は `arq summarize $ARGUMENTS` で生成を提案する。

### 3. 関連する精読済み論文の検索

`arq search` で論文のキーワードに関連する精読済み論文を探す。

```bash
arq search "<論文の主要キーワード>"
```

ヒットした論文の `reads/` レポートも読み込み、論文間の関連性や比較を踏まえた議論を可能にする。

### 4. ディスカッション

レポートと原文の内容を踏まえて、ユーザーと対話的に議論する。以下の観点で深堀りを促す：

- 手法の詳細や前提条件への疑問
- 関連論文との比較・位置づけ
- 自分の仕事（超伝導QCのシステム開発）への示唆
- 限界や未解決問題
- フォローアップすべき研究の方向性

**回答する際は必ずグラウンディングルールに従う。** 具体的には：

- 「Section X によると、...」のように引用元を明示する
- 質問に対する直接的な記述が論文にない場合は「この点について論文では直接言及されていません」と答える
- ユーザーの仕事への示唆など推論を含む場合は「論文の結果（Section X）を踏まえると、...と考えられます」のように事実と推論を分ける

### 5. コメントの追記

ディスカッションの内容を、arq 管理下の **note.md** の末尾に追記する。追記は **ユーザーが明示的に「追記して」「メモして」「記録して」等と指示した場合** に行う。

note.md のパスは `arq show --json $ARGUMENTS` の `note_path` から取得する。
note.md が存在しない場合は life/reads/ のレポートにフォールバックする。

```markdown

---

## ディスカッションノート

### YYYY-MM-DD: [テーマ]

- 議論の要点をバレットポイントで記録
- 根拠となるセクション・図表の参照を含める（例: cf. Section 3.2, Fig. 4）
- 関連論文への言及があれば arxiv ID 付きで
- 自分の仕事への具体的な示唆やアクションアイテム
```

複数回のディスカッションがある場合は、`### YYYY-MM-DD: [テーマ]` のブロックを追加していく。
